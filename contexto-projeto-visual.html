Estudo · pra depois

1 painel só, todos os clientes em servidores separados

Cada cliente roda solo no servidor dele, mas o dono entra em um lugar só e vê todos. Sem um link por cliente. A base (Neon central + sync) já existe — só falta distribuir.

Modelo recomendado — banco central + sync por servidorestende o atual
cada servidorroda o bot + o sync (empurra conversas/leads pro Neon central, no schema daquele cliente) + uma control API (pausar/responder) no domínio do cliente.
dashboard centralum registro de agentes {slug, schema, url da control API...}. Cadastra o cliente → aparece junto dos outros. Leitura toda do Neon central.
controlepausar/enviar roteia pra control API daquele servidor (o dashboard guarda a URL por agente).
isolamentoschema por cliente no Neon central — nunca mistura.

Descartado: 1 dashboard por cliente (vira mil links) · dashboard federado consultando cada servidor ao vivo (complexo/lento). Não implementar agora — é o caminho de escala; a base já está pronta.

Dashboard (mvp_agente_ia) — pronto

  • Visão geral ciente do agente (form / outreach / none) · Conversas (abas WhatsApp/E-mail + filtro Ativas 24h/Responderam + tag de origem).
  • Leads (disparar + Programar) · Campanhas (vars = array jsonb, nunca JSON.stringify) · Disparos (agendar + auto-recuperação).
  • Neon central (schema por agente + public.*), syncs no cron, deploy git→Vercel. Envs: DATABASE_URL, DASHBOARD_PASSWORD, PAINEL_API_TOKEN, DISPATCH_CRON_SECRET.
